Описание программы calculatingOptimalCoins
Назначение:
Программа предназначена для решения задачи оптимального расчёта монет при оплате покупки. Программа получает на вход сумму покупки и набор доступных номиналов монет, после чего вычисляет минимальное количество монет, необходимое для оплаты. Если точная оплата невозможна, программа находит вариант с минимальной сдачей.

Запуск программы
Программа принимает два параметра командной строки:
1. Путь к входному файлу - файл с данными о сумме покупки и номиналах монет
2. Путь к выходному файлу - файл для записи результата расчёта


Формат входных данных
Входной файл должен быть текстовым файлом с XML-разметкой, содержащим следующую структуру:
<config>
  <sum>сумма_покупки</sum>
  <nominals>
    <value>номинал_1</value>
    <value>номинал_2</value>
    ...
  </nominals>
</config>

Требования к входным данным:
Тег <config> - корневой элемент, обязательный
Тег <sum> - сумма покупки, целое число от 1 до 1000, обязательный
Тег <nominals> - контейнер для номиналов монет, обязательный
Тег <value> - номинал монеты, уникальное целое число от 1 до 100, не более 100 штук

Пример входного файла:
<config>
  <sum>20</sum>
  <nominals>
    <value>3</value>
    <value>6</value>
  </nominals>
</config>

Формат выходных данных
Выходной файл генерируется в формате XML со следующей структурой:
<config>
  <change>сдача</change>
  <nominals>
    <nominal>
      <value>номинал</value>
      <count>количество</count>
    </nominal>
    ...
  </nominals>
</config>

Описание элементов:
Тег <change> - величина сдачи (разность между суммой оплаты и суммой покупки)
Тег <nominals> - контейнер для использованных номиналов
Тег <nominal> - описание одного использованного номинала
Тег <value> - номинал монеты
Тег <count> - количество монет данного номинала

Пример выходного файла:
<config>
  <change>1</change>
  <nominals>
    <nominal>
      <value>6</value>
      <count>3</count>
    </nominal>
    <nominal>
      <value>3</value>
      <count>1</count>
    </nominal>
  </nominals>
</config>


Обработка ошибок
Программа реализует расширенную систему обработки ошибок, которая классифицирует и выводит сообщения о следующих типах ошибок:
Ошибки файлов:
- Файл не найден или недоступен для чтения/записи
- Пустой входной файл
- Невозможность создания выходного файла
Ошибки данных:
- Сумма покупки не является числом или целым числом
- Сумма покупки вне допустимого диапазона [1, 1000]
- Номиналы монет не являются числами или целыми числами
- Номиналы вне допустимого диапазона [1, 100]
- Повторяющиеся номиналы монет
- Превышение максимального количества номиналов (более 100)
Структурные ошибки XML:
- Неопознанные теги в XML-документе
- Отсутствующие обязательные теги (<config>, <sum>, <nominals>)
- Некорректное расположение тегов (нарушение иерархии)
- Некорректный синтаксис XML


Требования к программному обеспечению
Для запуска скомпилированной программы не требуется установка дополнительного программного обеспечения. Все необходимые библиотеки включены в папку Executable/. Для разработки и сборки проекта требуется установка Qt Creator с компонентом MinGW 64-bit.

Работа с файлами
Входные файлы должны быть подготовлены в любом текстовом редакторе (например, Блокнот Windows) с соблюдением XML-синтаксиса.
Выходные файлы могут быть просмотрены в любом текстовом редакторе.
Кодировка файлов: UTF-8.
